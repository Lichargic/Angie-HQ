# Security Policy

## What to keep private

If you find something security-related in Angie, please do not post the details in a public issue.

That includes vulnerabilities, credential leaks, exploit paths, private account details, or anything that could help someone abuse the site.

## How to report

Use GitHub's private vulnerability reporting flow if it is available for this repo. If GitHub does not show that option, open a minimal public issue saying you need a private security contact. Do not include reproduction steps, screenshots with private data, secrets, or exploit details in that public issue.

## What to include

The most useful private reports include:

- The affected Angie page or tool.
- What could go wrong if the issue is abused.
- Steps to reproduce, without unnecessary private data.
- Browser and device details if the issue only happens client-side.

Angie is a solo project, so I may not respond like a full security team. I will still treat security reports as more urgent than normal feedback.
