# Security Policy

## What counts as a security issue

Security issues are things that could let someone abuse or manipulate Angie in unintended ways. That includes cross-site scripting, script injection, API proxy abuse, or anything that could affect how the site behaves for other users.

Wrong NPC prices, stale Bazaar data, or calculation errors are not security issues. Use the [data issue template](https://github.com/Lichargic/Angie-HQ/issues/new?template=data_issue.yml) for those.

## How to report

Use GitHub's private vulnerability reporting. There is a "Report a security vulnerability" option when you open a new issue. This keeps the details private while I investigate.

Do not post reproduction steps, screenshots, or exploit details in a public issue.

## What to include

The most useful reports include:

- What could go wrong if the issue is abused
- The affected page or feature
- Steps to reproduce, without unnecessary private data
- Browser and device details if the issue only appears client-side

## What to expect

Angie is a solo project. I will acknowledge security reports as soon as I can and aim to fix confirmed issues before any public disclosure. I will keep you updated and work with you on timing before anything is made public.
